# 《万象净界：纪元王庭》技术设计文档

> 版本：0.1  
> 日期：2026-07-19  
> 范围：20–30 分钟 Web 垂直切片

## 1. 技术目标

垂直切片必须证明：DOM 界面、Canvas 2D 游戏、WebGL 2 Shader 演出、动态音频和按需加载小游戏可以在同一个页面中安全切换；刷新或异常恢复不丢进度；低配置与减少动态模式仍完整可玩。

## 2. 产品默认决策

- 工作名继续使用《万象净界：纪元王庭》。
- 主角是可命名的光影“拾界者”，使用高辨识度披风轮廓，首版不做复杂捏脸。
- 美术以程序化精致像素、2.5D 视差和少量 WebGL 2 演出为主。
- Chrome/Edge/Firefox/Safari 当前稳定版为目标；桌面优先，同时提供完整触屏操作。
- WebGL 2 是特效基线；WebGPU 只做能力检测与未来增强入口，不阻塞完整游戏。
- 不引入重叠的 2D 引擎。垂直切片采用自研小型 Canvas 2D 场景层，以控制包体和生命周期；后续 Alpha 再根据实测决定是否迁移 Phaser。

## 3. 分层

```text
DOM Shell
  ├─ HUD / 对话 / 设置 / 暂停 / 错误恢复 / 触控
  ├─ GameHost（唯一主循环）
  │   ├─ Scene（王庭、原初荒原、森语庭院）
  │   ├─ MiniGameHost（动态 import、统一生命周期）
  │   ├─ Canvas2DRenderer（世界与小游戏）
  │   └─ ShaderRenderer（净化、传送门、上下文恢复）
  ├─ Core
  │   ├─ Progression / Quest / Economy
  │   ├─ Input（键鼠、触屏、Gamepad）
  │   ├─ Save（IndexedDB + localStorage 回退）
  │   ├─ Audio（Web Audio 动态声部）
  │   └─ Capabilities / Quality / Diagnostics
  └─ Content（场景定义、对话、小游戏 Manifest）
```

## 4. 场景与主循环

- 全局只存在一个 `requestAnimationFrame` 主循环。
- Canvas 使用 1280×720 逻辑坐标，CSS 自适应缩放；不直接按最高 DPR 扩张显存。
- `GameHost` 每帧完成输入轮询、固定步长更新、渲染和诊断采样。
- 世界场景和小游戏实现相同的 `Playable` 生命周期：`enter / update / render / resize / dispose`。
- 场景切换先 `dispose` 旧模块，再动态导入新模块；失败时回到最近安全场景。

## 5. 小游戏 Manifest

每个小游戏必须声明：

```ts
interface MiniGameManifest {
  id: MiniGameId;
  title: string;
  realm: 'wasteland' | 'garden';
  estimatedMinutes: number;
  unlock: (state: GameState) => boolean;
  load: () => Promise<MiniGameFactory>;
  reward: RewardDefinition;
  accessibility: string[];
}
```

运行时通过 `MiniGameContext` 读取动作输入、设置、音频和随机种子；小游戏不能直接修改全局存档。它只能返回标准 `MiniGameResult`，由核心进度系统验证并结算。

## 6. 存档

- 当前 Schema 为 v2。
- IndexedDB 保存当前槽和最近三个恢复点；不可用时回退 localStorage。
- 每个重要里程碑写入：临时记录 → 校验 → 当前记录 → 备份轮转。
- 导入先解析、迁移、校验，成功后才替换当前存档。
- 保存内容仅包含确定性状态与安全场景坐标，不序列化 Canvas、音频或运行时对象。

## 7. 输入

统一动作：移动、主行动、次行动、闪避、切换透镜、伙伴能力、暂停。

- 键盘：WASD/方向键、空格/Enter、E、Shift、Q、Escape。
- 触屏：虚拟方向区、行动、闪避、透镜按钮。
- 手柄：标准 Gamepad 映射，轮询按钮边沿与摇杆死区。
- UI 根据最近一次输入来源切换提示。

## 8. 渲染与特效

- 世界和四个小游戏由 Canvas 2D 渲染，使用对象池粒子、视差层和轻量屏幕震动。
- `ShaderRenderer` 使用唯一 WebGL 2 上下文，实现程序噪声净化溶解和伪 3D 界门。
- Low 模式关闭 WebGL 后处理并使用 Canvas 径向光圈替代；High 模式开启 Shader。
- `prefers-reduced-motion` 或减少动态模式会关闭震屏、强视差和快速扭曲。
- WebGL context lost 时暂停 Shader、显示恢复提示；恢复后重新创建 Program/Buffer。

## 9. 音频

- 首次“唤醒世界”点击创建/恢复 `AudioContext`。
- 环境、节拍、和声、旋律四条程序化声部根据净化进度平滑混音。
- 音效使用短包络振荡器，避免首版音频素材包。
- 页面失焦时淡出并暂停；回到页面后在用户交互时恢复。
- 设置分别控制总音量、音乐和音效。

## 10. 能力检测和质量档

启动时检测 WebGL 2、WebGPU 入口、OffscreenCanvas、触控、Gamepad、设备内存提示和减少动态偏好。检测结果只用于选择安全默认值，不作为阻止进入游戏的理由。

- Low：30 FPS 可接受，低粒子、无 Shader、低视差。
- High：目标 60 FPS，完整粒子、WebGL 溶解、动态光晕。
- 自动策略连续低于目标帧率时降低粒子预算；不会自动改变玩法规则。

## 11. PWA 与离线

- Service Worker 缓存应用壳和构建资源。
- 安装新版本时保留旧页面，刷新后才切换；存档迁移在应用层完成。
- 离线仍可完整完成垂直切片；在线能力不是核心依赖。

## 12. 验证

- 单元测试：状态迁移、结果结算、存档校验、小游戏生命周期。
- E2E：新游戏首次净化、刷新继续、四小游戏调试入口、两种透镜、导入导出、设置持久化。
- 性能：构建体积、FPS 采样、连续切换场景后的诊断计数。
- 浏览器：Chromium 自动化；Firefox/Safari 形成手动矩阵和已知限制记录。

## 13. 退出标准

G1 退出需要：项目可构建；DOM/Canvas/WebGL/Audio 能切换；WebGL 关闭时完整可玩；小游戏释放后诊断计数归零；至少一组自动化证据覆盖存档迁移和资源生命周期。
