# 《万象净界：纪元王庭》项目目标

> 版本：0.1  
> 日期：2026-07-19  
> 当前阶段：垂直切片公开测试版<br>
> 试玩地址：https://jfliu3265.github.io/epoch-court/<br>
> 测试证据：[测试矩阵](./docs/测试矩阵.md)
> 配套文档：[产品需求文档](./docs/产品需求文档.md)

## North Star Goal

打造一款打开网页即可游玩、能长期扩展为“小游戏帝国”的原创幻想冒险游戏：玩家通过多个真正好玩的短局玩法净化界域、重建共同王庭，并在推进过程中持续解锁新的画面、声音、交互和世界规则。

项目的第一个真实目标不是一次性造出全部帝国，而是完成一个 20–30 分钟、可公开试玩的垂直切片，证明以下核心命题：

1. 世界的表现和规则会随玩家行动产生令人惊喜的永久演化；
2. 多个小游戏属于同一个世界，并共同推动王庭、故事和角色成长；
3. 已解锁的时代能力可以带回旧区域，产生新的路线与解法；
4. 炫酷 Web 特效在目标桌面和移动设备上可读、流畅且可以降级；
5. 新小游戏和新界域能通过统一模块协议继续加入，而不破坏存档和核心系统。

## 当前产品假设

- 玩家真正想要的是《Evoland／进化之地》式“媒介与玩法不断进化的惊喜”，而不是对原作内容的复刻。
- 玩家会因为王庭的永久变化和跨玩法能力联动，把小游戏理解成一个统一冒险，而不是小游戏菜单。
- 精致像素 + 2.5D + 少量高价值 3D，比全程重 3D 更适合 Web 的性能、包体和持续内容生产。
- WebGL 2 应是基础渲染路径；WebGPU 只做高端增强。
- 首版应单人、离线优先、无强制账号；实时多人和 MMO 化不属于当前目标。

这些假设必须通过原型和玩家测试验证，而不是被当成永久真理。

## 垂直切片交付物

垂直切片至少包含：

- 一个从 1-bit 单色无声世界开始的可玩序章；
- 一座会因玩家行动产生永久变化的王庭门厅；
- 原初荒原与森语庭院两个小型界域；
- 四个可完成、可失败、可重试、可结算的小游戏；
- 至少两次明显的画面/音频/规则演化；
- 至少两种可改变视觉和玩法的时代透镜；
- 一次“获得新能力后重返旧地图并发现新路线”的闭环；
- 王庭建设、统一资源、任务、奖励和角色对话；
- 本地自动存档、继续游戏、存档导入导出和迁移版本；
- 键鼠与触屏操作，低/高画质，减少动态、字幕和色盲辅助；
- 一个具有章节完成感的结尾，以及下一界域预告。

## 完成定义（Definition of Done）

只有同时满足以下条件，垂直切片才算完成：

### 体验

- 新玩家无需开发者口头解释，可以在 3 分钟内移动并完成第一次有效互动。
- 至少 70% 的测试玩家可以独立完成第一次净化。
- 玩家在体验结束后能准确描述“小游戏如何帮助王庭或世界恢复”。
- 前 15 分钟至少出现三次玩家能主动感知的世界演化。
- 四个小游戏都不只是技术 Demo：有清晰目标、反馈、失败、重试、结算与第二层挑战。

### 技术

- 当前稳定版 Chrome、Edge、Firefox、Safari 的目标路径通过冒烟测试。
- WebGPU 不可用时仍可完整游玩。
- 中档桌面 High 维持目标 60 FPS；目标移动设备 Medium 稳定 45–60 FPS，Low 不低于稳定 30 FPS。
- 连续进入和退出全部小游戏后，没有持续增长的 GPU 纹理、音频节点、事件监听或计时器。
- 刷新页面、关闭标签页、浏览器崩溃恢复和一次模拟版本升级不会丢失已确认进度。
- 加载失败、WebGL context lost 和音频暂停都有用户可理解的恢复路径。

### 质量

- 关键攻击、危险区域和可交互物在各种画质与色盲辅助下仍可辨认。
- 所有不可跳过的语音信息都有字幕；高强度动态和镜头晃动可关闭。
- 没有 P0/P1 阻断缺陷，没有已知存档损坏路径。
- 初始应用壳和首个可玩包满足产品需求文档中的包体预算，后续界域按需加载。

## 执行路线

### G0：锁定概念基线

目标：让团队对“做什么、不做什么、为什么”达成一致。

- [x] 完成 Web 游戏特效与能力边界调研。
- [x] 校正《Evoland／进化之地》灵感来源。
- [x] 完成产品需求文档初稿。
- [x] 完成项目目标与完成标准。
- [x] 原型阶段继续使用工作名“万象净界：纪元王庭”。
- [x] 锁定原型默认方向：光影拾界者、精致像素/2.5D、桌面优先兼容触屏。

退出条件：所有阻塞原型实现的产品问题都有默认答案。

### G1：技术尖峰原型

目标：用最小代码验证组合技术可行，而不是搭建完整框架。

- [x] 建立 TypeScript + Vite 项目与基础质量工具。
- [x] 验证 DOM UI、2D 游戏场景和 WebGL Shader 演出能安全切换。
- [x] 完成设备能力检测与 WebGL 2 基线、WebGPU 增强回退。
- [x] 完成一个净化溶解 Shader、一个粒子效果、一个动态音乐层。
- [x] 验证场景卸载、资源释放、标签页失焦与 WebGL context 恢复。
- [ ] 在至少一台中档电脑和一台中端手机建立性能基线（自动化视口通过，仍需真实设备长局采样）。

退出条件：技术选型有实测数据，资源不会因多次切换而持续泄漏。

### G2：第一场净化

目标：产出 10–15 分钟“世界苏醒”体验。

- [x] 完成单色序章、移动、互动和最小 HUD。
- [x] 完成第一次色彩与声音恢复。
- [x] 完成“史莱姆闪避场”小游戏。
- [x] 完成统一结算、资源奖励和王庭门厅变化。
- [x] 完成本地自动存档与刷新恢复。
- [ ] 进行 5 名新玩家可用性测试并记录问题。

退出条件：玩家理解“完成小游戏 → 净化世界 → 王庭成长”的第一层闭环。

### G3：时代能力与重访闭环

目标：证明项目不只是线性换皮，而有真正的规则演化。

- [x] 完成像素层和水彩层两种时代透镜。
- [x] 同一场景在不同透镜下改变可见物和互动规则。
- [x] 完成遗迹推箱和花粉连线小游戏。
- [x] 玩家获得第二种透镜后，可回原初荒原发现隐藏路线。
- [x] 完成时代谱系、单透镜插槽和对应教程。

退出条件：测试玩家能举出至少一个“新能力改变旧内容”的具体例子。

### G4：第二界域与内容管线

目标：证明新界域和小游戏可以规模化加入。

- [x] 完成森语庭院小型地图与独立美术/音频风格。
- [x] 完成萤火节奏小游戏。
- [x] 用统一小游戏 Manifest 接入四个玩法。
- [x] 完成内容包按需加载、失败恢复和卸载清理。
- [x] 完成触屏操作、低/高画质和基础无障碍设置。

退出条件：加入第二界域时不需要重写核心存档、输入、奖励或场景生命周期。

### G5：垂直切片发布候选

目标：把原型整理成可以交给陌生玩家的完整作品。

- [x] 串联目标时长内容、角色对白、章节结尾与下一界域预告（真实玩家时长待可用性测试确认）。
- [x] 完成加载、教程、设置、暂停、失败、恢复和退出体验。
- [x] 建立浏览器、设备、性能、存档迁移和可访问性测试矩阵。
- [ ] 进行至少 20 名目标玩家测试。
- [x] 自动化覆盖范围内无 P0/P1 阻断缺陷；完成定义中的真人/真机条目仍待验证。
- [x] 发布可访问 URL 和版本化构建：https://jfliu3265.github.io/epoch-court/

退出条件：本文件“完成定义”全部满足，并形成是否进入 Alpha 的书面决策。

### G6：Alpha 帝国扩张

仅在 G5 通过后启动：

- [ ] 扩展至三个完整界域、八个小游戏和一个可建设城区。
- [ ] 完成伙伴系统、界域信任、精通挑战和收藏图鉴。
- [ ] 建立可重复的关卡、对话、奖励和小游戏制作模板。
- [ ] 根据数据决定是否加入账号、云存档和异步挑战。

## 优先级规则

当需求冲突时，按以下顺序决策：

1. 存档安全与能否正常进入游戏；
2. 操作响应、战斗/谜题可读性与稳定帧率；
3. 核心循环是否完整、小游戏是否真正属于共同世界；
4. 无障碍、输入覆盖与错误恢复；
5. 画面精度和额外特效；
6. 内容数量；
7. 在线、社交与商业化能力。

任何新增内容如果让当前里程碑无法形成闭环，应推迟到后续里程碑。

## 明确的非目标

当前项目不追求：

- 复刻《Evoland》或任何现有游戏的角色、剧情、美术和关卡；
- 在第一个版本制作 30 个小游戏或 9 个完整界域；
- 实时 MMO、公会、聊天、开放世界服务器；
- 抽卡、体力、强制广告和付费战力；
- 必须使用最新 GPU 才能进入的技术展示；
- 为了内容数量接受不可维护的独立小游戏代码；
- 牺牲清晰操作和性能来换取不可关闭的全屏特效。

## 工程守则

- 游戏规则与渲染分离；核心状态不依赖具体 UI 框架。
- 小游戏通过标准事件结算，不直接修改全局存档。
- 所有持久化结构必须带版本，并有迁移测试。
- 所有重型资源都有所有者、生命周期和释放验证。
- 所有效果都有 Low 档或关闭路径，核心提示不能只依赖颜色。
- 所有随机玩法使用可记录的种子，便于复现错误和未来异步挑战。
- 每个里程碑必须在真实目标设备上测试，不能只看开发机。
- 先 Profile 再优化；性能预算和包体预算在合并内容时检查。

## 下一步

公开测试版已经完成实现、自动化验证与发布。进入 Alpha 之前只剩外部验证门槛：

1. 组织 5 名新玩家完成首次净化测试并记录问题；
2. 扩展至 20 名目标玩家，验证理解度、完成率与 20–30 分钟目标时长；
3. 在中档电脑、中端手机、Firefox、Safari、Edge 上执行测试矩阵并记录 FPS 与兼容问题；
4. 根据真实反馈形成是否进入 G6 Alpha 帝国扩张的书面决策。
